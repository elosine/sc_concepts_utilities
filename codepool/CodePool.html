CodePool a system for collaborative coding


Inherits from: Object


CodePool was designed to be used by a group to perform collaborative works. Additionally it is well suited to situations where you simply want to keep track of what environmental variables are in use and what/when code has been compiled and by whom.


This class is cross platform with some exceptions. Currently the Swing version:

- does not create the other post window showing everything as it comes in.

- does not show timestamps on code block as Windows SC doesn't have Date working.

- does not let you send your code by pressing return, you need to click the button.


Some things to keep in mind when using CodePool:

- make sure to .store (not .send!) all of your SynthDefs in the CodePool window before using them so that everyone will have them.

- only "~" variables will appear in the variable list so it is best to avaoid sending code with non "~" variables.

- as this is ment to be a collaborative system, it is a good idea to send your code in format readable by humans at a glance.



Creation / Class Methods


*new (name, addr,port)

Create a new CodePool instance.

name - A String identifying code sent from this instance. Default is "Default".

addr - The String IP address to send code to. Default is "127.0.0.1".

port - The int port to send code to. Default is 57120.

// example

c = CodePool.new("me", "127.0.0.1", 57120);

c.putCode("\"test\""); // writes "test" to the code window

c.sendCode; // same as pressing the "Send" button or Enter in the code window



Accessing Instance and Class Variables

codeArray

Array containing all code that has been sent using CodePool. Used to get specific blocks of code that has been sent if you know the index. This is mainly here for cases where you would want to automate CodePool.

Default value is [].

name

Explanation including the type of name and a link to its help file.

Default value is Default.

port

Port of where to send code to.

Default value is 57120.

Default value is "127.0.0.1".

send (code)

Sends a String of code to the specified network address and port. Once sent it will compile on the target machine.

code - Any valid SuperCollider code written as a String in quotes. to send code involving quotes make sure to use \".

// send some codeÂ 

s = Server.local;

s.boot;

c = CodePool.new("me", "127.0.0.1", 57120);

c.send("a = {SinOsc.ar(440,0,0.3)}.play;"); //code must be enclosed in quotes ""

c.send("a.release(1);");

c.send("\"test\""); // make sure to use \" for quotes



putCode (code)

Puts a String of code to the code window without sending it anywhere. This is mainly here for cases where you would want to automate CodePool.

code - Any valid SuperCollider code written as a String in quotes. to send code involving quotes make sure to use \".

// put some code in the code window

c.putCode("\"test\""); // watch the code window



getCode

Returns the current contents of the code window as a String. This is mainly here for cases where you would want to automate CodePool. For access to specific blocks of code use codeArray.

// store the contents of the code window into a variable

s = c.getCode; // get the code window as a String (make sure something is there first)

s.postln;



sendCode

Sends the current contents of the code window to the specified network address and port. This is mainly here for cases where you would want to automate CodePool.

// make sure there's something in the code window

c.sendCode;
